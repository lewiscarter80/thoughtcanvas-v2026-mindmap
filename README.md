## What is ThoughtCanvas?

ThoughtCanvas is an open-source desktop application for creating mind maps without a cloud dependency or mandatory account sign-in. Its local-first design keeps your work on your own device, so you can continue editing even when you are offline.

The app is centered on adaptable planning and visual thinking. It supports brace maps, spider-web maps, more than one canvas in a single file, and extended topic content. For people who want a practical mapping environment with structure, notes, and an open JSON-based format, ThoughtCanvas offers a focused workspace.

## Key capabilities

- Handles several map styles, including brace maps and spider-web maps
- Runs as a desktop application and is intended for offline, local-first use
- Stores data in an open format using plain JSON and .bmap files
- Lets you keep multiple canvases in one file
- Includes AI-assisted creation and editing for maps
- Provides an outline view with two-way sync
- Includes a Gantt view for timeline-oriented planning
- Accepts .bmap files through drag and drop
- Supports UI personalization with skins, colors, and styles
- Offers keyboard shortcuts to speed up editing
- Supports rich topics with tags, notes, links, images, and task details

## How to use it

Begin with a new canvas, then pick the structure that matches your brainstorming, planning, or research process. You can create a map from scratch, drag a `.bmap` file into the app to import it, or return to a file you already started.

Typical workflow:
1. Open or create a map file.
2. Add topics and connect ideas using the preferred map structure.
3. Switch to outline view when you want a linear editing pass.
4. Use Gantt view when the work needs a timeline.
5. Add notes, tags, links, images, and task data to individual topics.
6. Adjust the look and feel with a skin or custom color scheme.

## Configuration

ThoughtCanvas stores most project data in the map file itself, which fits its local-file workflow. Appearance settings and workspace behavior are expected to be handled from inside the app and through file-based usage.

Example of the kind of file-oriented structure you may work with:

    {
      "canvases": [],
      "maps": [],
      "theme": "custom",
      "format": "json"
    }

Use the built-in customization options to change skins, colors, and layout behavior as needed.
